# タイミングエディタ

ステージ楽曲の「音ハメ」ポイントにマーカーを打ち、Claude に渡す指示書 Markdown を
書き出すためのツールです。任意のステージで使えます。

## 起動方法

`index.html` をブラウザ（Chrome / Edge 推奨）で開くだけ。サーバは不要です。
音源や画像はローカルで処理され、外部に送信されません。

## 基本フロー

1. **音源を読み込む** … 「音源を選択」ボタン、またはタイムラインへ音声ファイルをドロップ。
   （石工ステージは「標準音源」ボタンで `Assets/StageData/stone/stone.m4a` を読み込めます）
2. **ステージ設定を入力** … ツールバーの以下を埋める:
   - **曲名**（例: 石工）… タイトルと指示書見出しに使用
   - **作曲者**（任意）… 空なら指示書の「（作曲: …）」は省略
   - **スラッグ**（例: stone、半角英数とハイフンのみ）… ファイル名・自動保存キーに使用
   - **BPM / 拍子 / 拍オフセット** … 波形上の拍グリッド基準。拍オフセットで小節頭を合わせる
3. **再生しながら打刻** … `Space` で再生し、要所で `K` を押してマーカーを追加。
   波形上の位置は右パネルの「小節:拍」「拍ジャスト / 拍から ±ms」で確認できます。
4. **メモを記入** … 右パネルでラベル・コメントを編集。`I` で選択マーカーに画像を添付できます。
5. **書き出す** … 右下パネルで「Claude向け」を選び「MD保存」または「コピー」。
   JSON でマーカー一式を保存/復元することもできます（「保存」/「JSON読込」）。
6. **リポジトリへ保存** … 書き出した Markdown を `Instructions/{slug}-timing-instructions.md`
   として保存し、Claude に渡します（画像はチャットに別途添付）。

## キーバインド

キー操作はテキスト入力欄にフォーカスしていないときに有効です。

| キー | 動作 |
|---|---|
| `Space` | 再生 / 停止 |
| `K` | 再生位置にマーカー追加（コメント欄にフォーカス） |
| `I` | 選択マーカーに画像を添付 |
| `Delete` / `Backspace` | 選択マーカーを削除 |
| `[` / `]` | 選択マーカーを −10ms / +10ms 微調整 |
| `←` / `→` | タイムラインを横スクロール（`Shift` で大きく移動） |
| `+` / `=` / `-` | ズームイン / ズームアウト |

マウス操作:

- **クリック**: その位置へシーク　/　**Shift+クリック**: その位置にマーカー追加
- **ドラッグ**・**ホイール**: タイムライン横スクロール　/　**Ctrl（⌘）+ホイール**: ズーム
- **マーカーをドラッグ**: 時刻を移動　/　**再生ヘッド付近をドラッグ**: スクラブ
- **一覧の行をクリック**: 選択　/　**ダブルクリック**: その時刻へジャンプ

## 自動保存

- 編集内容（マーカー・設定）は **スラッグ単位** で `localStorage` に自動保存されます
  （変更の約1秒後）。次回起動時に、前回のスラッグのデータを自動復元します。
- スラッグを別の値に変えると、そのスラッグの保存データがあれば読み込むか確認します。
  無ければ現在の内容をそのまま新しいスラッグで保存し続けます。
- **音源ファイル自体は保存されません。** 復元後は音源を読み込み直してください。
- 別ステージ／別 PC へ引き継ぐときは JSON で書き出して共有します。

## 指示書の書式について

- `秒` 列は Unity 側 `BulletBuffer` の発火 / `appearTime` の基準に対応させる値です。
- `小節:拍` は BPM グリッド上の位置（1始まり）、`間隔` は直前マーカーからの差（秒 / 拍数）。
- 表・見出しの書式は Claude への指示パイプラインが依存しているため、そのまま渡してください。
